# 第 7 项：接口层编码

## 1. 目标

接口层负责接收浏览器请求，完成参数接收、基础校验、登录状态判断、页面跳转、数据返回和异常处理。接口层不直接编写数据库操作逻辑，应通过业务逻辑层完成具体功能。

接口层主要完成以下工作：

- 定义系统页面路由；
- 定义前端 AJAX 请求接口；
- 接收表单参数和查询参数；
- 调用 Service 完成业务处理；
- 将业务结果放入 `Model`、`Session` 或 JSON 响应中；
- 根据用户身份控制页面访问范围；
- 为前端页面提供统一、清晰的接口说明。

## 2. 技术路线

系统采用 Spring Boot + Spring MVC + Thymeleaf 的接口层实现方式。

整体调用链如下：

```text
浏览器页面
    -> Controller 接口层
    -> Service 业务逻辑层
    -> Mapper 数据访问层
    -> 数据库
```

接口层分为两类：

| 类型 | 作用 | 返回形式 |
|---|---|---|
| 页面路由接口 | 打开页面、分页展示、查询结果页 | Thymeleaf 模板名称 |
| 数据操作接口 | 登录、添加、删除、修改、借书、还书 | JSON、布尔值或重定向 |

页面展示类接口适合使用 `@Controller`，数据返回类接口可使用 `@ResponseBody` 返回 JSON 数据。

## 3. 接口层目录结构

推荐结构如下：

```text
src/main/java/com/zbw/controller
├── LoginController.java
├── UserController.java
├── AdminController.java
├── BookController.java
├── BookCategoryController.java
└── BorrowingBooksController.java
```

各 Controller 的职责如下：

| Controller | 职责 |
|---|---|
| `LoginController` | 用户登录、管理员登录、退出登录 |
| `UserController` | 用户首页、用户信息、用户新增、用户删除、用户分页 |
| `AdminController` | 管理员首页、管理员信息修改、图书录入、分类维护 |
| `BookController` | 图书查询、按分类分页查询、借书页面 |
| `BookCategoryController` | 图书分类列表、分类新增、分类删除 |
| `BorrowingBooksController` | 借书、还书、个人借阅记录、全部借阅记录 |

## 4. 接口层编码原则

接口层应遵循以下规则：

1. Controller 只负责请求处理，不写复杂业务逻辑；
2. Controller 不直接调用 Mapper；
3. 登录态统一保存在 Session 中；
4. 用户接口和管理员接口应区分访问权限；
5. 页面跳转接口返回模板名称；
6. AJAX 接口返回 JSON 或布尔结果；
7. 所有关键参数必须进行空值和格式校验；
8. 删除、修改、借书、还书等操作应使用 POST 请求；
9. 分页参数应设置默认值，避免空页码或非法页码导致异常；
10. 异常场景应返回明确结果，避免前端无法判断操作状态。

## 5. 登录接口设计

### 5.1 用户登录

用户登录接口接收用户名和密码，调用用户业务逻辑完成校验。登录成功后，将用户对象保存到 Session，并跳转到用户首页；登录失败时返回登录页并给出失败标记。

接口定义：

| 项目 | 内容 |
|---|---|
| 请求路径 | `/userLogin` |
| 请求方式 | POST |
| 请求参数 | `userName`、`password` |
| 调用业务 | `IUserService.userLogin(userName, password)` |
| 成功结果 | 保存 `user` 到 Session，跳转用户首页 |
| 失败结果 | 保存失败标记，返回登录页 |

示例代码：

```java
@PostMapping("/userLogin")
public String userLogin(String userName, String password,
                        HttpServletRequest request) {
    User user = userService.userLogin(userName, password);

    if (user == null) {
        request.getSession().setAttribute("flag", true);
        return "index";
    }

    request.getSession().setAttribute("user", user);
    request.getSession().removeAttribute("flag");
    return "user/index";
}
```

### 5.2 管理员登录

管理员登录接口与用户登录接口类似，但调用管理员业务逻辑，并将管理员对象保存到 Session。

接口定义：

| 项目 | 内容 |
|---|---|
| 请求路径 | `/adminLogin` |
| 请求方式 | POST |
| 请求参数 | `userName`、`password` |
| 调用业务 | `IAdminService.adminLogin(userName, password)` |
| 成功结果 | 保存 `admin` 到 Session，跳转管理员首页 |
| 失败结果 | 保存失败标记，返回登录页 |

示例代码：

```java
@PostMapping("/adminLogin")
public String adminLogin(String userName, String password,
                         HttpServletRequest request) {
    Admin admin = adminService.adminLogin(userName, password);

    if (admin == null) {
        request.getSession().setAttribute("flag", true);
        return "index";
    }

    request.getSession().setAttribute("admin", admin);
    request.getSession().removeAttribute("flag");
    return "admin/index";
}
```

### 5.3 退出登录

退出登录时应清除 Session 中的登录对象。

| 请求路径 | 请求方式 | 处理逻辑 | 返回结果 |
|---|---|---|---|
| `/userLogOut` | GET | 清除 `user` | 登录页 |
| `/adminLogOut` | GET | 清除 `admin` | 登录页 |

示例代码：

```java
@GetMapping("/userLogOut")
public String userLogOut(HttpServletRequest request) {
    request.getSession().removeAttribute("user");
    return "index";
}
```

## 6. 用户页面接口

用户页面接口主要为普通用户提供首页、个人信息、图书查询、借书、还书和借阅记录页面。

| 请求路径 | 请求方式 | 功能 | 返回页面 |
|---|---|---|---|
| `/userMessagePage` | GET | 查看个人信息 | `user/userMessage` |

| `/userBorrowBookRecord` | GET | 查看个人借阅记录 | `user/borrowingBooksRecord` |

个人借阅记录接口需要从 Session 中获取当前登录用户，再调用业务逻辑查询该用户的借阅记录。

示例代码：

```java

    model.addAttribute("borrowingBooksList", list);

}
```

## 12. 接口参数校验规则

接口层需要完成基础参数校验，复杂业务校验交给 Service 层处理。

| 参数 | 校验规则 |
|---|---|
| `userName` | 非空，长度不超过用户表字段限制 |
| `password` | 非空，长度不超过用户表字段限制 |
| `userEmail` | 非空时应符合邮箱格式 |
| `adminEmail` | 非空时应符合邮箱格式 |
| `bookId` | 非空，正整数 |
| `userId` | 非空，正整数 |
| `bookCategoryId` | 非空，正整数 |
| `bookCategory` | 非空，正整数 |
| `bookName` | 非空 |
| `bookPrice` | 不能为负数 |
| `pageNum` | 为空或小于 1 时按第 1 页处理 |

示例：

```java
private int normalizePageNum(Integer pageNum) {
    if (pageNum == null || pageNum < 1) {
        return 1;
    }
    return pageNum;
}
```

## 13. 登录状态与权限控制

普通用户和管理员应使用不同的 Session 键保存登录对象。

| 身份 | Session 键 | 可访问模块 |
|---|---|---|
| 普通用户 | `user` | 图书查询、借书、还书、个人信息、个人借阅记录 |
| 管理员 | `admin` | 图书管理、用户管理、分类管理、全部借阅记录、管理员信息 |

接口层应在进入受保护页面前检查登录状态。

示例：

```java
private boolean isUserLogin(HttpServletRequest request) {
    return request.getSession().getAttribute("user") != null;
}
```

如果未登录，应返回登录页或跳转到登录页。

## 14. 返回结果规范

接口层返回结果分为页面返回和 AJAX 返回两类。

### 14.1 页面返回

页面接口返回 Thymeleaf 模板名称。

示例：

```java
return "admin/showUsers";
```

页面数据通过 `Model` 传递。

示例：

```java
model.addAttribute("page", page);
```

### 14.2 AJAX 返回

AJAX 接口返回布尔值、数字或 JSON 对象。

| 返回值 | 含义 |
|---|---|
| `true` | 操作成功 |
| `false` | 操作失败 |
| JSON 数组 | 查询结果列表 |
| JSON 对象 | 结构化返回数据 |

推荐统一返回结构：

```json
{
  "success": true,
  "message": "操作成功",
  "data": {}
}
```

若继续使用布尔值返回，也应保证前端能够明确区分成功和失败。

## 15. 异常处理

接口层应避免异常直接暴露给浏览器。

常见异常处理规则如下：

| 异常场景 | 处理方式 |
|---|---|
| 未登录访问用户页面 | 返回登录页 |
| 未登录访问管理员页面 | 返回登录页 |
| 参数为空 | 返回失败结果或默认值 |
| 数字参数非法 | 返回失败结果 |
| 数据不存在 | 返回空列表或失败结果 |
| 写操作失败 | 返回 `false` |
| Service 抛出异常 | 记录日志并返回失败结果 |

示例：

```java
@PostMapping("/deleteUser")
@ResponseBody
public boolean deleteUser(Integer userId) {
    if (userId == null || userId <= 0) {
        return false;
    }
    return userService.deleteUserById(userId) > 0;
}
```

## 18. 接口层测试建议

接口层测试应覆盖页面接口和 AJAX 接口。

| 测试类型 | 测试内容 | 预期结果 |
|---|---|---|
| 页面访问测试 | 访问登录页、用户页、管理员页 | 页面正常渲染 |
| 登录测试 | 输入正确和错误账号密码 | 成功进入首页，失败返回登录页 |
| 参数校验测试 | 传入空 `bookId`、非法 `pageNum` | 返回失败或默认页 |
| AJAX 测试 | 新增、删除、修改、借书、还书 | 前端能收到明确结果 |
| 权限测试 | 未登录访问用户或管理员页面 | 返回登录页 |
| 分页测试 | 点击上一页、下一页 | 页码和数据正确变化 |
| 异常测试 | 删除不存在用户、借阅不存在图书 | 返回失败结果 |

可使用 Spring Boot Test 和 MockMvc 编写接口层测试。

示例：

```java
@RunWith(SpringRunner.class)
@SpringBootTest
@AutoConfigureMockMvc
public class LoginControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    public void userLoginSuccess() throws Exception {
        mockMvc.perform(post("/userLogin")
                .param("userName", "user1")
                .param("password", "123456"))
                .andExpect(status().isOk());
    }
}
```
