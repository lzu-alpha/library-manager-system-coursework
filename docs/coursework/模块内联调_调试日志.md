## 4. 问题记录与处理

| 编号 | 问题现象 | 原因分析 | 处理方式 | 状态 |
|---|---|---|---|---|
| D-01 | 测试环境需要数据库支持 | 自动化联调不能依赖外部数据库服务 | 使用 H2 内存数据库并导入测试表结构和测试数据 | 已解决 |
| D-02 | MyBatis SQL 与测试数据库语法存在兼容风险 | 原 SQL 面向 MySQL，测试库需要兼容 MySQL 方言 | 使用 H2 MySQL 兼容模式 | 已解决 |
| D-03 | 借书和还书测试会修改借阅记录 | 写操作会影响后续测试数据 | 在详细业务测试中使用事务回滚 | 已解决 |
| D-04 | 仅输出控制台内容不能证明业务正确性 | 原测试以打印为主，缺少断言 | 增加断言型详细业务测试 | 已解决 |
| D-05 | 前端接口依赖 `true` / `false` 判断操作结果 | 返回值不统一时前端不易处理 | 联调文档中规定 AJAX 接口返回明确布尔值或统一 JSON | 已规范 |
