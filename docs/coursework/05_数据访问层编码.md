# 第 5 项：数据访问层编码

## 1. 目标

完成系统数据访问层代码，实现对后台数据库的增删改查、条件查询和分页查询，并通过单元测试验证数据库操作的正确性。

数据访问层主要承担以下职责：

- 定义数据库访问接口；
- 编写 MyBatis XML 映射文件；
- 完成实体类与数据库表字段映射；
- 实现基础 CRUD 操作；
- 实现业务所需的扩展查询；
- 编写数据访问层单元测试。

## 2. 数据访问层结构

数据访问层由三部分组成：

| 部分 | 目录 | 作用 |
|---|---|---|
| 实体类 | `src/main/java/com/zbw/domain` | 对应数据库表结构 |
| Mapper 接口 | `src/main/java/com/zbw/mapper` | 定义数据库访问方法 |
| Mapper XML | `src/main/resources/mapper` | 编写 SQL 映射语句 |

基本关系如下：

```text
Controller
    ↓
Service
    ↓
Mapper 接口
    ↓
Mapper XML
    ↓
MySQL 数据库
```

Controller 和 Service 不直接编写 SQL，所有数据库访问统一通过 Mapper 层完成。

## 3. 数据表与 Mapper 对应关系

系统主要数据表与 Mapper 的对应关系如下：

| 数据表 | 实体类 | Mapper 接口 | Mapper XML |
|---|---|---|---|
| `admin` | `Admin` | `AdminMapper` | `AdminMapper.xml` |
| `user` | `User` | `UserMapper` | `UserMapper.xml` |
| `dept` | `Department` | `DepartmentMapper` | `DepartmentMapper.xml` |
| `book_category` | `BookCategory` | `BookCategoryMapper` | `BookCategoryMapper.xml` |
| `book` | `Book` | `BookMapper` | `BookMapper.xml` |
| `borrowingbooks` | `BorrowingBooks` | `BorrowingBooksMapper` | `BorrowingBooksMapper.xml` |

每张表应至少具备对应的实体类、Mapper 接口和 XML 映射文件。

## 4. 实体类设计

实体类用于承载数据库表中的数据。实体类属性采用小驼峰命名，数据库字段采用下划线命名，两者通过 MyBatis `resultMap` 映射。

示例：用户表实体类

```java
public class User {
    private Integer userId;
    private String userName;
    private String userPwd;
    private String userEmail;
}
```

示例：图书表实体类

```java
public class Book {
    private Integer bookId;
    private String bookName;
    private String bookAuthor;
    private String bookPublish;
    private Integer bookCategory;
    private Double bookPrice;
    private String bookIntroduction;
}
```

实体类设计要求：

1. 属性名称应能准确表达字段含义；
2. 属性类型应与数据库字段类型匹配；
3. 实体类不编写业务流程；
4. 实体类只承担数据承载职责；
5. 对外提供必要的 getter 和 setter。

## 5. Mapper 接口设计

Mapper 接口用于定义数据库操作方法。接口方法应命名清晰，并与 XML 中的 SQL ID 保持一致。

### 5.1 基础 CRUD 方法

每个核心 Mapper 应提供基础增删改查方法。

示例：

```java
public interface UserMapper {
    int insert(User record);

    int insertSelective(User record);

    int deleteByPrimaryKey(Integer userId);

    int updateByPrimaryKey(User record);

    int updateByPrimaryKeySelective(User record);

    User selectByPrimaryKey(Integer userId);

    List<User> selectByExample(UserExample example);
}
```

方法说明：

| 方法 | 作用 |
|---|---|
| `insert` | 插入完整记录 |
| `insertSelective` | 插入非空字段 |
| `deleteByPrimaryKey` | 根据主键删除 |
| `updateByPrimaryKey` | 根据主键更新完整记录 |
| `updateByPrimaryKeySelective` | 根据主键更新非空字段 |
| `selectByPrimaryKey` | 根据主键查询 |
| `selectByExample` | 根据条件对象查询 |

### 5.2 扩展查询方法

除基础 CRUD 外，还应根据业务需求增加扩展查询方法。

示例：按图书名称模糊查询

```java
List<Book> selectBooksByPartInfo(String bookPartInfo);
```

示例：按分类分页查询图书

```java
List<Book> selectByCategoryId(@Param("categoryId") int categoryId,
                              @Param("currIndex") int currIndex,
                              @Param("pageSize") int pageSize);
```

示例：查询某一分类下图书总数

```java
int selectBookCountByCategoryId(@Param("categoryId") int categoryId);
```

示例：分页查询借阅记录

```java
List<BorrowingBooks> selectAllByPage(@Param("currIndex") int currIndex,
                                     @Param("pageSize") int pageSize);
```

扩展查询方法应满足：

- 方法名称体现查询条件；
- 多参数方法使用 `@Param` 标注；
- 返回值类型明确；
- 查询逻辑写在 XML 中；
- 不在 Mapper 中处理业务规则。

## 6. MyBatis XML 编写规范

### 6.1 XML 基本结构

Mapper XML 文件应包含 `namespace`、`resultMap`、字段列表和 SQL 语句。

示例：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.zbw.mapper.BookMapper">

    <resultMap id="BaseResultMap" type="com.zbw.domain.Book">
        <id column="book_id" jdbcType="INTEGER" property="bookId"/>
        <result column="book_name" jdbcType="VARCHAR" property="bookName"/>
        <result column="book_author" jdbcType="VARCHAR" property="bookAuthor"/>
        <result column="book_publish" jdbcType="VARCHAR" property="bookPublish"/>
        <result column="book_category" jdbcType="INTEGER" property="bookCategory"/>
        <result column="book_price" jdbcType="DOUBLE" property="bookPrice"/>
        <result column="book_introduction" jdbcType="VARCHAR" property="bookIntroduction"/>
    </resultMap>

</mapper>
```

说明：

- `namespace` 必须与 Mapper 接口全限定名一致；
- `resultMap` 用于映射数据库字段和 Java 属性；
- 主键字段使用 `<id>`；
- 普通字段使用 `<result>`。

### 6.2 通用字段列表

推荐将常用字段抽取为 SQL 片段，减少重复。

示例：

```xml
<sql id="Base_Column_List">
    book_id, book_name, book_author, book_publish,
    book_category, book_price, book_introduction
</sql>
```

查询时使用：

```xml
<include refid="Base_Column_List"/>
```

### 6.3 新增操作

完整新增：

```xml
<insert id="insert" parameterType="com.zbw.domain.Book">
    insert into book (
        book_id, book_name, book_author,
        book_publish, book_category, book_price,
        book_introduction
    )
    values (
        #{bookId}, #{bookName}, #{bookAuthor},
        #{bookPublish}, #{bookCategory}, #{bookPrice},
        #{bookIntroduction}
    )
</insert>
```

选择性新增：

```xml
<insert id="insertSelective" parameterType="com.zbw.domain.Book">
    insert into book
    <trim prefix="(" suffix=")" suffixOverrides=",">
        <if test="bookName != null">book_name,</if>
        <if test="bookAuthor != null">book_author,</if>
        <if test="bookPublish != null">book_publish,</if>
    </trim>
    <trim prefix="values (" suffix=")" suffixOverrides=",">
        <if test="bookName != null">#{bookName},</if>
        <if test="bookAuthor != null">#{bookAuthor},</if>
        <if test="bookPublish != null">#{bookPublish},</if>
    </trim>
</insert>
```

### 6.4 删除操作

根据主键删除：

```xml
<delete id="deleteByPrimaryKey" parameterType="java.lang.Integer">
    delete
    from book
    where book_id = #{bookId}
</delete>
```

删除操作应注意外键约束。如果被删除的数据已被其他表引用，需要先处理关联数据或明确级联策略。

### 6.5 修改操作

根据主键更新：

```xml
<update id="updateByPrimaryKey" parameterType="com.zbw.domain.Book">
    update book
    set book_name = #{bookName},
        book_author = #{bookAuthor},
        book_publish = #{bookPublish},
        book_category = #{bookCategory},
        book_price = #{bookPrice},
        book_introduction = #{bookIntroduction}
    where book_id = #{bookId}
</update>
```

选择性更新：

```xml
<update id="updateByPrimaryKeySelective" parameterType="com.zbw.domain.Book">
    update book
    <set>
        <if test="bookName != null">book_name = #{bookName},</if>
        <if test="bookAuthor != null">book_author = #{bookAuthor},</if>
        <if test="bookPublish != null">book_publish = #{bookPublish},</if>
    </set>
    where book_id = #{bookId}
</update>
```

### 6.6 查询操作

根据主键查询：

```xml
<select id="selectByPrimaryKey" parameterType="java.lang.Integer" resultMap="BaseResultMap">
    select
    <include refid="Base_Column_List"/>
    from book
    where book_id = #{bookId}
</select>
```

模糊查询：

```xml
<select id="selectBooksByPartInfo" parameterType="java.lang.String" resultMap="BaseResultMap">
    select
    <include refid="Base_Column_List"/>
    from book
    where book_name like #{bookPartInfo}
</select>
```

分页查询：

```xml
<select id="selectByCategoryId" resultMap="BaseResultMap">
    select
    <include refid="Base_Column_List"/>
    from book
    where book_category = #{categoryId}
    limit #{currIndex}, #{pageSize}
</select>
```

## 7. 参数绑定规范

SQL 参数应使用 `#{}` 绑定，避免 SQL 注入。

推荐：

```xml
where user_name = #{userName}
```

不推荐：

```xml
where user_name = '${userName}'
```

多参数 Mapper 方法应使用 `@Param`：

```java
List<User> selectByPageNum(@Param("currIndex") int currIndex,
                           @Param("pageSize") int pageSize);
```

XML 中使用对应参数名：

```xml
limit #{currIndex}, #{pageSize}
```

## 8. 分页查询实现

分页查询通常需要两个 SQL：

1. 查询当前页数据；
2. 查询总记录数。

示例：分页查询用户

```java
List<User> selectByPageNum(@Param("currIndex") int currIndex,
                           @Param("pageSize") int pageSize);

int selectUserCount();
```

XML 示例：

```xml
<select id="selectByPageNum" resultMap="BaseResultMap">
    select *
    from user
    limit #{currIndex}, #{pageSize}
</select>

<select id="selectUserCount" resultType="java.lang.Integer">
    select count(*)
    from user
</select>
```

分页参数计算：

```java
int currIndex = (pageNum - 1) * pageSize;
```

分页结果由统一分页对象保存：

```java
public class Page<T> {
    private List<T> list;
    private int pageNum;
    private int pageSize;
    private int pageCount;
}
```

## 9. 数据访问层单元测试

数据访问层完成后，需要编写单元测试验证数据库操作。

### 9.1 测试范围

应覆盖以下内容：

- 根据主键查询；
- 条件查询；
- 模糊查询；
- 分页查询；
- 新增数据；
- 修改数据；
- 删除数据；
- 查询总数；
- 外键关联数据查询。

### 9.2 测试类命名

测试类命名格式：

```text
实体名 + MapperTest
```

示例：

```text
UserMapperTest
BookMapperTest
BookCategoryMapperTest
BorrowingBooksMapperTest
```

### 9.3 测试示例

```java
@RunWith(SpringRunner.class)
@SpringBootTest
public class BookMapperTest {

    @Resource
    private BookMapper bookMapper;

    @Test
    public void testSelectByPrimaryKey() {
        Book book = bookMapper.selectByPrimaryKey(1);
        assertNotNull(book);
        assertEquals(Integer.valueOf(1), book.getBookId());
    }

    @Test
    public void testSelectBooksByPartInfo() {
        List<Book> books = bookMapper.selectBooksByPartInfo("%Java%");
        assertNotNull(books);
    }
}
```

### 9.4 测试要求

1. 测试方法应使用断言判断结果；
2. 不应只使用 `System.out.println` 输出；
3. 测试数据应可重复执行；
4. 涉及新增、修改、删除的测试应清理测试数据；
5. 测试数据库应与正式数据库区分；
6. 测试失败时应能明确定位到具体 SQL 或 Mapper 方法。

## 10. 数据访问层常见问题

### 10.1 Mapper 方法找不到

可能原因：

- Mapper 接口方法名与 XML 中的 SQL ID 不一致；
- XML 的 `namespace` 与 Mapper 接口路径不一致；
- XML 文件未放在 `resources/mapper` 目录；
- `mapper-locations` 配置错误。

处理方式：

- 检查接口方法名和 XML ID；
- 检查 `namespace`；
- 检查 XML 文件路径；
- 检查 MyBatis 配置。

### 10.2 查询结果字段为空

可能原因：

- 数据库字段名与实体属性名不一致；
- `resultMap` 映射错误；
- SQL 查询字段不完整；
- 实体类属性类型不匹配。

处理方式：

- 检查 `resultMap`；
- 检查字段列表；
- 检查实体类属性类型；
- 检查数据库表结构。

### 10.3 分页结果不正确

可能原因：

- `currIndex` 计算错误；
- `pageSize` 参数错误；
- 总记录数 SQL 与分页 SQL 条件不一致；
- 页码小于 1。

处理方式：

- 使用 `(pageNum - 1) * pageSize` 计算起始位置；
- 校验页码参数；
- 保证分页查询和总数查询条件一致。

### 10.4 插入或删除失败

可能原因：

- 外键约束冲突；
- 必填字段为空；
- 主键重复；
- 字段长度超过限制；
- 数据类型不匹配。

处理方式：

- 检查数据库约束；
- 检查实体对象字段；
- 检查 SQL 参数；
- 捕获异常并记录日志。

## 11. 数据访问层提交检查清单

提交数据访问层代码前，应检查以下内容：

| 检查项 | 要求 |
|---|---|
| 实体类 | 属性与数据库字段对应 |
| Mapper 接口 | 方法命名清晰，参数明确 |
| XML namespace | 与 Mapper 接口全限定名一致 |
| resultMap | 字段映射完整正确 |
| SQL 参数 | 使用 `#{}` 绑定 |
| CRUD | 增删改查方法可用 |
| 分页 | 当前页查询和总数查询配套 |
| 单元测试 | 覆盖核心查询和写操作 |
| 异常风险 | 外键、空值、重复主键已考虑 |
| 编码 | XML 和 SQL 文件使用 UTF-8 |
