# 第 1 项：开发环境配置手册

## 1. 目标

本项说明开发环境配置需要完成的工作。完成配置后，开发人员应能够：

- 使用 IDE 正确导入项目源码；
- 初始化 MySQL 数据库；
- 修改后端数据库连接配置；
- 理解项目的基础目录结构；
- 明确启动和访问系统所需的环境条件。

## 2. 项目技术栈概述

本项目是一个基于 Java Web 技术的图书管理系统，主要技术组成如下：

| 类型 | 技术 |
|---|---|
| 后端语言 | Java |
| 后端框架 | Spring Boot 风格工程 |
| 数据访问 | MyBatis |
| 数据库 | MySQL |
| 页面模板 | Thymeleaf |
| 前端交互 | jQuery、Layui |
| 测试框架 | JUnit、Spring Boot Test |

项目采用典型分层结构：实体类、数据访问层、业务逻辑层、页面模板、静态资源和数据库脚本分别放在不同目录中，便于后续开发和维护。

## 3. 开发环境准备

开发人员需要提前安装以下软件：

| 软件 | 建议版本 | 用途 |
|---|---|---|
| JDK | JDK 8 或以上 | 编译和运行 Java 后端代码 |
| IntelliJ IDEA / Eclipse | 较新稳定版本 | 导入、编辑和运行项目 |
| MySQL | MySQL 5.7 或兼容版本 | 存储系统业务数据 |
| Git | 较新稳定版本 | 代码版本管理 |
| 数据库管理工具 | Navicat、DataGrip、MySQL Workbench 等 | 执行 SQL 脚本、查看数据表 |

建议所有源代码文件、SQL 文件、HTML 文件和配置文件统一使用 UTF-8 编码，避免中文注释、页面文字或初始化数据出现乱码。

## 4. 项目目录说明

导入项目后，开发人员需要重点关注以下目录和文件：

| 路径 | 作用 |
|---|---|
| `src/main/java` | Java 后端源码目录 |
| `src/main/java/com/zbw/domain` | 实体类和查询条件类 |
| `src/main/java/com/zbw/mapper` | MyBatis Mapper 接口 |
| `src/main/java/com/zbw/service` | 业务逻辑接口 |
| `src/main/java/com/zbw/service/impl` | 业务逻辑实现类 |
| `src/main/java/com/zbw/utils` | 工具类目录 |
| `src/main/resources/application.yml` | 系统主要配置文件 |
| `src/main/resources/mapper` | MyBatis XML 映射文件 |
| `src/main/resources/db` | 数据库初始化脚本 |
| `src/main/resources/templates` | Thymeleaf 页面模板 |
| `src/main/resources/static` | CSS、JavaScript、图片等静态资源 |
| `src/test/java` | 测试代码目录 |

第一次接手项目时，应先确认上述目录是否完整，再进行数据库和运行环境配置。

## 5. IDE 导入与编码设置

### 5.1 导入项目

1. 打开 IntelliJ IDEA 或 Eclipse。
2. 选择项目根目录进行导入。
3. 将 `src/main/java` 设置为源码目录。
4. 将 `src/main/resources` 设置为资源目录。
5. 将 `src/test/java` 设置为测试源码目录。

如果 IDE 没有自动识别项目结构，需要手动设置源码目录和资源目录。

### 5.2 设置文件编码

在 IDE 中将项目编码设置为 UTF-8：

- Global Encoding：UTF-8；
- Project Encoding：UTF-8；
- Properties Files Encoding：UTF-8。

编码统一后，可以减少配置文件、页面、Java 注释和 SQL 初始化数据中的中文乱码问题。

## 6. 数据库配置

### 6.1 创建数据库

本项目使用 MySQL 数据库，数据库名称为：

```text
library-manager-system
```

首次配置时，需要先在 MySQL 中创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS `library-manager-system`
  DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;
```

创建完成后切换到该数据库：

```sql
USE `library-manager-system`;
```

### 6.2 执行初始化脚本

数据库初始化脚本位于：

```text
src/main/resources/db/library-manager-system.sql
```

可以通过数据库管理工具打开该 SQL 文件并执行，也可以在 MySQL 命令行中执行：

```sql
SOURCE src/main/resources/db/library-manager-system.sql;
```

执行完成后，应能看到系统所需的数据表，例如：

- `admin`
- `user`
- `dept`
- `book_category`
- `book`
- `borrowingbooks`

这些表用于保存管理员、用户、部门、图书分类、图书信息和借阅记录。

## 7. 后端配置文件说明

系统主要配置文件位于：

```text
src/main/resources/application.yml
```

### 7.1 服务端口配置

```yaml
server:
  port: 8080
```

说明：

- 系统启动后默认监听 `8080` 端口；
- 本地访问地址通常为 `http://localhost:8080`。

### 7.2 MyBatis 配置

```yaml
mybatis:
  mapper-locations: classpath:mapper/*.xml
  type-aliases-package: com.zbw.domain
```

说明：

- `mapper-locations` 表示 MyBatis XML 文件位于 `resources/mapper` 目录；
- `type-aliases-package` 表示实体类位于 `com.zbw.domain` 包；
- Mapper 接口与 XML 文件需要保持方法名和 SQL 映射 ID 一致。

### 7.3 数据库连接配置

```yaml
spring:
  datasource:
    driverClassName: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/library-manager-system?useUnicode=true&serverTimezone=UTC&characterEncoding=utf-8&useSSL=false
    username: root
    password: 123456
```

说明：

- `driverClassName` 为 MySQL JDBC 驱动；
- `url` 中的数据库名必须与实际创建的数据库一致；
- `username` 和 `password` 需要修改为本机 MySQL 的实际账号和密码；
- 如果 MySQL 端口不是 `3306`，也需要同步修改连接地址。

## 8. 项目依赖说明

项目运行通常需要以下依赖：

- Spring Boot Web；
- Spring Boot Test；
- MyBatis；
- MySQL Connector/J；
- Thymeleaf；
- Lombok；
- JUnit。

如果项目根目录中已经存在 Maven 或 Gradle 构建文件，应直接通过 IDE 自动导入依赖。

如果项目根目录中缺少 `pom.xml` 或 Gradle 构建文件，则需要先补充依赖管理文件，否则 IDE 可能无法正确识别 Spring Boot、MyBatis、MySQL 驱动、Lombok 和测试相关类。

## 9. 启动与访问

完成数据库初始化和配置文件修改后，可以尝试启动后端应用。

启动成功后，在浏览器访问：

```text
http://localhost:8080
```

如果启动失败，应依次检查：

1. JDK 是否配置正确；
2. 项目依赖是否完整；
3. MySQL 服务是否已启动；
4. 数据库 `library-manager-system` 是否已创建；
5. SQL 初始化脚本是否已执行；
6. `application.yml` 中的数据库用户名和密码是否正确；
7. 端口 `8080` 是否被其他程序占用；
8. 项目是否存在 Spring Boot 启动类。

## 10. 常见问题与处理

### 10.1 数据库连接失败

可能原因：

- MySQL 服务未启动；
- 数据库名写错；
- 用户名或密码不正确；
- MySQL 端口不是 `3306`；
- JDBC 驱动依赖缺失。

处理方式：

- 启动 MySQL 服务；
- 确认数据库名为 `library-manager-system`；
- 修改 `application.yml` 中的用户名和密码；
- 检查项目依赖中是否包含 MySQL Connector/J。

### 10.2 页面或注释出现中文乱码

可能原因：

- IDE 项目编码不是 UTF-8；
- SQL 文件导入时编码不一致；
- 数据库字符集不是 UTF-8。

处理方式：

- 将 IDE 项目编码设置为 UTF-8；
- 使用 UTF-8 方式重新导入 SQL；
- 确认数据库字符集为 `utf8` 或 `utf8mb4`。

### 10.3 Mapper XML 无法加载

可能原因：

- `mapper-locations` 配置错误；
- XML 文件没有放在 `resources/mapper` 目录；
- Mapper 接口方法名与 XML 中的 SQL ID 不一致。

处理方式：

- 确认配置为 `classpath:mapper/*.xml`；
- 确认 XML 文件位于 `src/main/resources/mapper`；
- 检查 Mapper 接口和 XML 文件中的方法映射是否一致。

### 10.4 IDE 无法识别项目依赖

可能原因：

- 缺少 Maven 或 Gradle 构建文件；
- IDE 没有正确导入依赖；
- 本地依赖下载失败。

处理方式：

- 补充 `pom.xml` 或 Gradle 构建文件；
- 使用 IDE 重新导入项目；
- 检查 Maven 或 Gradle 本地配置。
