# 第 8 项：模块内联调

## 1. 目标

模块内联调用于验证数据访问层、业务逻辑层和接口调用契约能否正确衔接。前面各层代码分别完成后，还需要通过联调确认各模块之间的调用关系、参数传递、返回结果、异常处理和页面数据组织是否一致。

本项重点完成以下工作：

- 联调数据访问层与业务逻辑层；
- 检查 Service 调用 Mapper 的结果是否正确；
- 检查业务方法返回值能否满足接口层使用要求；
- 检查分页数据、VO 对象和 Session 数据是否能被页面正确使用；
- 发现并记录接口调用中的问题；
- 形成联调报告和调试日志。

## 2. 联调范围

本次联调覆盖系统核心模块。

| 模块 | 联调内容 |
|---|---|

## 4. 模块调用关系

系统模块内部调用关系如下：

```text
页面请求
    -> Controller
    -> Service
    -> Mapper
    -> Database
```

各层职责如下：

| 层次 | 职责 |
|---|---|
| Controller | 接收请求、校验参数、调用 Service、返回页面或 JSON |
| Service | 处理业务规则、组织 VO、控制事务、调用 Mapper |
| Mapper | 执行 SQL，完成数据库增删改查 |
| Database | 保存用户、管理员、图书、分类、借阅记录等数据 |

联调重点是检查相邻层之间的数据传递是否一致。

## 6. 联调流程

模块内联调按以下顺序执行：

1. 启动测试环境；
2. 加载 Spring Boot 应用上下文；
3. 初始化测试数据库表结构；
4. 导入测试数据；
5. 测试 Mapper 是否可以正常执行 SQL；
6. 测试 Service 是否可以正常调用 Mapper；
7. 测试 Service 是否能正确组织分页对象和 VO 对象；
8. 测试借书、还书等写操作是否符合业务规则；
9. 检查事务回滚是否生效；
10. 记录联调结果和问题处理过程。

## 8. 接口调用契约联调

接口层需要接收页面请求并调用业务层。联调时重点检查接口参数与 Service 方法参数是否一致。

| 前端操作 | 接口路径 | 关键参数 | Service 方法 | 返回要求 |
|---|---|---|---|---|
| 用户登录 | `/userLogin` | `userName`、`password` | `userLogin` | 成功进入用户首页，失败返回登录页 |
| 管理员登录 | `/adminLogin` | `userName`、`password` | `adminLogin` | 成功进入管理员首页，失败返回登录页 |
| 查询图书 | `/findBookByBookPartInfo` | `bookPartInfo` | `selectBooksByBookPartInfo` | 返回 `bookList` |
| 借书 | `/userBorrowingBook` | `bookId` | `userBorrowingBook` | 返回 `true` 或 `false` |
| 还书 | `/userReturnBook` | `bookId` | `userReturnBook` | 返回 `true` 或 `false` |
| 查询分类 | `/findAllBookCategory` | 无 | `getBookCategories` | 返回分类 JSON 数组 |
| 新增图书 | `/addBook` | 图书表单字段 | `addBook` | 返回成功或失败 |
| 新增分类 | `/addBookCategory` | `categoryName` | `addBookCategory` | 返回成功或失败 |
| 删除分类 | `/deleteCategory` | `bookCategoryId` | `deleteBookCategoryById` | 返回成功或失败 |
| 删除用户 | `/deleteUser` | `userId` | `deleteUserById` | 返回成功或失败 |
| 查看借阅记录 | `/allBorrowBooksRecordPage` | `pageNum` | `selectAllByPage` | 返回 `page` |

接口调用契约检查结果：

- 参数名称与前端表单字段保持一致；
- 借书和还书接口使用 `bookId` 作为统一参数；
- 分页接口使用 `pageNum` 作为统一页码参数；
- AJAX 操作接口需要返回前端可判断的结果；
- 页面展示接口需要向 `Model` 中放入页面模板使用的数据名称。
