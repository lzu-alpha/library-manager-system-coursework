## 11. 事务日志

详细业务测试中，写操作被事务包裹。

日志摘录：

```text
Began transaction (1) for test context
Registering transaction synchronization for SqlSession
JDBC Connection ... will be managed by Spring
Releasing transactional SqlSession
Transaction synchronization deregistering SqlSession
Transaction synchronization closing SqlSession
Rolled back transaction for test
```

事务行为分析：

| 行为 | 说明 |
|---|---|
| `Began transaction` | 测试方法开始时开启事务 |
| `Registering transaction synchronization` | MyBatis SqlSession 绑定到当前事务 |
| `will be managed by Spring` | JDBC 连接交由 Spring 事务管理 |
| `Releasing transactional SqlSession` | SQL 执行后释放当前事务中的会话引用 |
| `Rolled back transaction` | 测试方法结束后回滚写操作 |

结论：借书和还书测试不会污染后续测试数据，联调结果可重复执行。

## 12. SqlSession 生命周期日志

非事务查询场景：

```text
Creating a new SqlSession
SqlSession ... was not registered for synchronization because synchronization is not active
JDBC Connection ... will not be managed by Spring
Closing non transactional SqlSession
```

事务查询场景：

```text
Creating a new SqlSession
Registering transaction synchronization for SqlSession
JDBC Connection ... will be managed by Spring
Fetched SqlSession ... from current transaction
Releasing transactional SqlSession
Transaction synchronization closing SqlSession
```

分析：

- 普通查询测试没有显式事务，SqlSession 执行完后直接关闭；
- 详细业务测试启用事务，SqlSession 绑定到事务上下文；
- 写操作在事务中执行并最终回滚。
